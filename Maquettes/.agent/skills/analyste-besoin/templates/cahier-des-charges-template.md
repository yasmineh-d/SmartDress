# Capacité : Rédaction Cahier des Charges

## Objectif
Standardiser la rédaction du document `cahier-des-charges.md` qui fait foi pour tout le projet.

## Template Obligatoire

```markdown
# Cahier des Charges : [Nom du Projet]

## 1. Contexte & Enjeux
- **Pourquoi ce projet ?** : [Réponse]
- **Cible** : [Persona]
- **Tons & Style** : [Serieux/Ludique/Corporate...]

## 2. Besoins Fonctionnels
- **Besoin Métier** : [Quel problème résout-on ?]
- **Fonctionnalités Clés** : 
    - [Action] (ex: Présenter les services)
    - [Action] (ex: Contact par email/réseaux)
- **Contenus Indispensables** : [Liste des contenus requis sans structurer les pages]

## 3. Contraintes
- **Deadline** : [Date/Durée]
- **Technique** : Site Statique (Imposé)
- **Langue** : Français
```

## Règles de Rédaction
1. Utiliser le **Markdown** pur.
2. Être concis (Bullet points).
3. Ne pas utiliser de jargon technique complexe, le client doit comprendre.
