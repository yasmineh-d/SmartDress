1. Palette de couleurs

La palette SmartDress est inspirée des matières naturelles et organiques. Elle transmet une identité chaleureuse, professionnelle et raffinée, adaptée à une application de mode.

🟫 ContrasteBark Brown#5C4A35Textes principaux, header sombre, footer
🟡 SecondaireTan#CFBB99Hover states, badges, éléments décoratifs

Règles d'utilisation des couleurs

  • Moss Green est réservé aux éléments interactifs et aux mises en avant (CTA, liens actifs, tags de section).
  • Bark Brown assure le contraste texte/fond. Il est utilisé sur fond clair uniquement.
  • Tan et Bone servent de fonds secondaires, jamais comme couleur de texte principal.
  • Off-White est le fond de base de toutes les pages.
  • Ne jamais utiliser deux couleurs de fond proches sans séparateur visuel.
  • Le ratio de contraste texte/fond doit respecter les normes WCAG AA minimum.

2. Typographie

SmartDress utilise un système de deux polices complémentaires : une police d'affichage élégante pour les titres, et une police de corps lisible pour le contenu.

Polices

Police d'affichage — Cormorant Garamond

Famille  : Cormorant Garamond
Type     : Serif · Display
Source   : Google Fonts
Graisses : Light 300 · Regular 400 · SemiBold 600
Styles   : Normal · Italic
Usage    : Titres Hero, H1, H2 de section, noms de fonctionnalités, citations

Police de corps — DM Sans

Famille  : DM Sans
Type     : Sans-Serif · Humanist
Source   : Google Fonts
Graisses : Light 300 · Regular 400 · Medium 500
Usage    : Navigation, boutons, corps de texte, labels, captions

Règles typographiques

  • Les titres en Cormorant Garamond peuvent utiliser l'italique pour les mots clés (<em>).
  • L'interligne standard est 1.8 pour le corps de texte, 1.05 pour les grands titres.
  • L'espacement entre lettres (letter-spacing) des labels est 0.25em minimum.
  • Ne jamais utiliser Arial, Roboto ou Inter dans l'interface SmartDress.
  • Les titres en italique utilisent la couleur Moss Green pour l'emphase.

3. Iconographie

Style d'icônes

SmartDress utilise des icônes émoji natives pour garantir la lisibilité universelle et la légèreté visuelle. Ce choix favorise la cohérence multiplateforme sans dépendance à une bibliothèque externe.

Icônes principales de l'application

IcôneContexte d'utilisation
👗Garde-robe / Vêtements
🌤️Météo / Conditions climatiques
✨Suggestion IA / Mise en avant
🔔Notification / Alerte
📊Statistiques / Tableau de bord
🔒Sécurité / Confidentialité
👋Accueil / Salutation utilisateur
⛅Météo partiellement nuageux

Règles d'usage des icônes

  • Taille recommandée : 1.25rem à 1.75rem (20–28px)
  • Conteneur : Carré arrondi avec border-radius: 10–12px, fond Moss Green ou Cream
  • Alignement : Toujours centré horizontalement et verticalement dans son conteneur
  • Ne jamais placer une icône sans label textuel accessible (aria-label ou texte visible)
  • Les icônes dans les boutons sont positionnées à gauche du texte, séparées de 8px

6. Logo & Usage

Construction du logo

Le logo SmartDress est un logotype typographique composé de deux mots aux styles contrastés :

Smart     →  Cormorant Garamond  |  Regular 400  |  couleur Bark #5C4A35
Dress     →  Cormorant Garamond  |  SemiBold 600 Italic  |  couleur Moss #889063

Variantes

VarianteFondCouleur "Smart"Couleur "Dress"
PrincipaleOff-White / ClairBark #5C4A35Moss #889063
Fond sombreBark #5C4A35Cream #F5EEE4Tan #CFBB99

Zone de protection

La zone de protection autour du logo est égale à 0.5x la hauteur totale du logo sur chaque côté. Aucun élément graphique ne doit pénétrer dans cette zone.

Taille minimale

ContexteTaille minimale
Écran digital24px de hauteur
Impression12mm de hauteur
Favicon / icône app16×16px (version monogramme SD)

✅ Usages autorisés

  • Utiliser Smart (regular) + Dress (bold italic) — toujours ensemble
  • Conserver le rapport de tailles entre les deux mots
  • Utiliser exclusivement les couleurs de la charte
  • Respecter la zone de protection à chaque usage
  • Le monogramme SD peut être utilisé en version réduite (favicon, badge)

❌ Usages interdits

  • Étirer, déformer ou pivoter le logo
  • Changer les couleurs hors palette officielle
  • Placer le logo sur un fond qui ne garantit pas un contraste suffisant
  • Séparer Smart de Dress dans deux espaces ou lignes différentes
  • Ajouter des effets (ombre portée, contour, dégradé) au logotype
  • Modifier l'espacement entre les lettres ou entre les deux mots

7. Règles générales

Cohérence visuelle

  • Toujours utiliser les variables CSS (:root) pour référencer les couleurs — jamais de valeurs hardcodées dans les composants.
  • Les animations doivent rester subtiles : durée entre 0.25s et 0.7s, easing ease ou ease-out.
  • Les micro-interactions (hover, focus) doivent donner un retour visuel immédiat.
  • Privilégier les transitions CSS aux animations JavaScript quand possible.

Accessibilité

  • Contraste minimum texte/fond : 4.5:1 (WCAG AA) pour le corps de texte, 3:1 pour les grands titres.
  • Tous les éléments interactifs doivent avoir un état :focus visible.
  • Les icônes seules doivent avoir un attribut aria-label.
  • Les couleurs ne doivent jamais être le seul moyen de transmettre une information.

Responsive

  • L'interface est conçue en mobile-first : les styles de base ciblent les petits écrans.
  • Les grilles passent progressivement de 1 colonne (mobile) à 3 colonnes (desktop).
  • La navigation mobile utilise le composant Preline Collapse pour le menu hamburger.
  • Les éléments décoratifs complexes (ex: phone mockup) peuvent être masqués sur mobile.

Ton & Voix

  • L'interface parle en français, tutoiement ou vouvoiement selon le contexte (vouvoiement recommandé pour l'interface principale).
  • Le ton est élégant, chaleureux et rassurant — jamais technique ou froid.
  • Les labels de boutons sont en MAJUSCULES, courts et actifs : COMMENCER, ESSAYER, VOIR LA DÉMO.
  • Les descriptions sont fluides, à la 3ème personne du singulier : "SmartDress organise...", "L'application suggère...".
